API ENDPOINTS
================================================================================

HEALTH & STATUS:
  GET  /api/v1/health           - Health check
  GET  /api/v1/status           - Overall system status
  GET  /api/v1/status/history   - Uptime history (30-90 days)

INCIDENTS:
  GET  /api/v1/incidents        - Get all incidents
  POST /api/v1/incidents        - Create new incident (admin)
  GET  /api/v1/incidents/{id}   - Get specific incident
  POST /api/v1/incidents/{id}/update - Update incident (admin)

METRICS:
  GET  /api/v1/metrics          - All metrics with history
  GET  /api/v1/metrics/summary  - Metrics summary

SUBSCRIPTIONS:
  POST /api/v1/subscribe        - Subscribe to status updates
  POST /api/v1/unsubscribe      - Unsubscribe from updates

RSS:
  GET  /api/v1/status/rss       - RSS feed (XML)

================================================================================
DEPLOYMENT
================================================================================

DOCKER DEPLOYMENT:
  1. Configure environment variables in docker-compose.yml
  2. Run: docker-compose up -d
  3. Access: http://localhost:8080
  4. Check logs: docker-compose logs -f

PRODUCTION DEPLOYMENT:
  1. Set environment variables
  2. Build Docker image: docker build -t paw-status .
  3. Configure Nginx reverse proxy
  4. Enable HTTPS/SSL
  5. Set up monitoring for status page itself
  6. Configure email SMTP settings
  7. Set webhook URLs for notifications
